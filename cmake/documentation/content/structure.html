Project Structure

Once you generate a project using CMake_Build.bat, the folder layout will be organized like this:

Root/
 ├── .bin/                      # Output directory for built DLLs
 ├── .lib/                      # Output directory for static libraries
 ├── build/                     # CMake build folder (contains generated solution)
 ├── cmake/                     # Utility scripts and dependency configuration
 ├── CMake_Build.bat            # Bootstrap script to configure/regenerate project
 ├── CMakePresets.json          # Presets for debug/release and toolchain support
 ├── .gitignore                 # Git exclusions
 ├── CMakeLists.txt             # Root-level CMake configuration
 ├── MyPluginProject/           # Folder named after your plugin project
 │   ├── extern/                # External libraries (e.g. CommonLibSSE, ImGui, JSON)
 │   ├── include/               # Header files (*.h)
 │   ├── src/                   # Source files (*.cpp)
 │   ├── pch/                   # Precompiled headers (pch.h / pch.cpp)
 │   ├── resource/              # Resources (e.g. plugin.rc, plugin.def)

Where to Add Your Code

  • Header Files: Add them to MyPluginProject/include/
  • Source Files: Add them to MyPluginProject/src/
  • Resource Files: Use MyPluginProject/resource/ for .rc, .def, icons, etc.
  • Precompiled Headers: Modify MyPluginProject/pch/pch.h and pch.cpp if needed

Adding New Files

After adding new .h or .cpp files, re-run CMake_Build.bat to regenerate your Visual Studio solution and ensure your files are included in the build.

Visual Studio Filters

Filters in Solution Explorer are auto-generated based on your folder structure under include/ and src/, so everything stays organized automatically.
