Configuration

All project setup and dependency handling is managed through CMake, controlled by configuration files:

  • project_config.cmake – Main project metadata and CMake options
  • submodule_config.cmake – Controls which libraries and dependencies are included

CommonLibSSE Forks

This template supports two major forks of CommonLibSSE:

  • PO3: Popular and stable fork used by many community plugins
  • NG: Experimental fork with Anniversary Edition support and extended features

The fork you select determines which headers, versioning macros, and compatibility behaviors are activated during project generation.

Switching Between CommonLibSSE Forks

If you want to switch your project from one CommonLibSSE fork to another (for example, from PO3 to NG or vice versa), follow these steps carefully to ensure a clean transition:

  1. Delete the build/ folder to remove generated build files.
  2. Delete the extern/ folder inside your project directory to remove all pulled dependencies.
  3. Edit the CMakeLists.txt file and set the COMMONLIB_VARIANT option to your desired fork, for example:
    set(COMMONLIB_VARIANT "vs2022-po3") or set(COMMONLIB_VARIANT "vs2022-ng")
  4. Re-run CMake_Build.bat to regenerate the project and download the appropriate dependencies.

This process ensures that your project is fully rebuilt with the correct fork and dependencies without conflicts.
